## 3. ТИПЫ УЗЛОВ

Каждый узел имеет тип. Тип определяет цвет, поведение, описание.

```typescript
type NodeType = 
  | 'user'        // Пользователь — белый #e0e0e0
  | 'switch'      // Коммутатор L2 — серый #5a7090
  | 'router'      // Маршрутизатор L3 — синий #00b4ff
  | 'isp'         // ISP роутер — синий #0080cc
  | 'tspу'        // ТСПУ/DPI — красный #ff4444
  | 'vpn'         // VPN сервер — фиолетовый #9c6bff
  | 'firewall'    // Фаервол — оранжевый #ff8c00
  | 'cdn'         // CDN нода — зелёный #00e676
  | 'webserver'   // Веб-сервер — светло-серый #b0b0b0
  | 'dns-stub'    // DNS stub резолвер — янтарный #ffb300
  | 'dns-r'       // DNS рекурсивный — янтарный #ffb300
  | 'dns-root'    // DNS root серверы — тёмно-жёлтый #cc8800
  | 'dns-tld'     // DNS TLD (.com .ru) — жёлтый #ddaa00
  | 'dns-auth'    // DNS авторитетный — жёлто-зелёный #aacc00
```

### Конфиг узла (nodeConfig.ts)

```typescript
interface NodeConfig {
  type: NodeType
  label: string          // короткая метка внутри квадрата (макс 4 символа)
  fullName: string       // полное название
  color: string          // hex цвет рамки и буквы
  layer: string          // 'L2' | 'L3' | 'L4' | 'L7'
  canDo: string[]        // список возможностей (зелёные галочки в панели)
  cannotDo: string[]     // список ограничений (красные крестики)
  protocols: string[]    // протоколы которые обрабатывает
  description: string    // краткое описание для tooltip
  details: {             // подробное описание по полям
    status: string
    level: string
    function: string
    threat?: string
    protection?: string
    mechanic?: string[]
    bonus?: string
  }
}
```

## 4. ТИПЫ РЁБЕР

```typescript
interface Edge {
  id: string
  source: string         // id исходного узла
  target: string         // id целевого узла
  bandwidth: number      // Мбит/с (100, 1000, 10000)
  latency: number        // мс
  packetLoss: number     // % (0.0 - 100.0)
  utilization: number    // % текущей загрузки (динамически меняется)
  type: 'ethernet' | 'fiber' | 'vpn-tunnel' | 'wireless'
  label?: string         // подпись на ребре (например "WAN")
  bidirectional: boolean // двунаправленное или нет
}
```

### Визуал рёбер

```
Толщина линии:
  100 Мбит/с  → 1px
  1 Гбит/с   → 2px
  10 Гбит/с  → 4px

Цвет по утилизации:
  0-30%   → #1e2d4a  (тёмно-синий, спокойно)
  30-60%  → #00b4ff  (синий, активно)
  60-80%  → #ffb300  (янтарный, нагрузка)
  80-100% → #ff4444  (красный, перегрузка + потери пакетов)

Стиль линии:
  ethernet/fiber  → сплошная
  vpn-tunnel      → пунктир (#9c6bff цвет)
  wireless        → пунктир (#00e676 цвет)
```

```typescript
}
```

## 7. ФИКСИРОВАННАЯ ТОПОЛОГИЯ (topology.ts)

Это данные для режима TOPOLOGY. Не меняй координаты без необходимости.

```
ПУТЬ ДАННЫХ (горизонтально):
User-1 (100,200) → Switch (260,420) → Router-CPE (400,420)
User-2 (100,420) ↗                                ↓
User-3 (100,640) ↗                          Router-ISP (540,420)
                                                   ↓
                                             ТСПУ (680,420)
                                            ↙         ↘
                                    VPN (760,280)    прямой путь
                                         ↓                ↓
                                   Firewall (900,420) ←──┘
                                   ↙        ↓         ↘
                            WS1(1080,200) WS2(1080,420) WS3(1080,640)
                            news.com      google.com    blocked.com⚠

DNS ЦЕПОЧКА (внизу):
User-2 → DNS-Stub(260,780) → DNS-R(440,780) → DNS-ROOT(620,900)
                                                    ↓
                                              DNS-TLD(800,900)
                                             ↙            ↘
                                    DNS-AUTH1(980,780)  DNS-AUTH2(980,900)
                                    ns1.google.com      ns1.blocked.com
```

### Правила блокировки ТСПУ

```
WS3 (blocked.com) — ЗАБЛОКИРОВАН
  Прямой путь User→Switch→ISP→ТСПУ→FW→WS3 = BLOCKED
  Пакет разбивается на ТСПУ с анимацией осколков
  
  Обход через VPN:
  User→Switch→ISP→ТСПУ→VPN→FW→WS3 = TUNNELED (проходит)
  Пакет меняет цвет на фиолетовый при входе в VPN
```

## 8. АНИМАЦИИ

### Пакеты

```
Нормальный полёт:
  duration = 1500мс / speed
  easing = linear
  размер = 6x6px

При паузе:
  пакеты замирают на месте
  размер увеличивается до 8x8px
  появляется пульсирующий outline

Блокировка на ТСПУ:
  пакет разбивается на 4 осколка (4x квадрата 3x3px)
  осколки разлетаются в 4 стороны на 20px
  fadeOut за 400мс
  ТСПУ мигает красным 300мс
  лог: "✕ BLOCKED: User→blocked.com | SNI detected"

Шифрование VPN:
  при входе в VPN-ребро пакет меняет цвет на #9c6bff
  появляется внешний outline (имитация обёртки)
  при выходе из VPN-ребра outline исчезает

Bounce при отправке из User:
  узел User: scale 1.0 → 1.2 → 1.0 за 200мс
  пакет стартует с easeOut

DNS цепочка bounce:
  каждый DNS узел: scale 1.0 → 1.15 → 1.0 за 150мс
  с задержкой 150мс между узлами
```

## 10. OSPF РЕЖИМ

Активируется кнопкой [OSPF].

```
При включении:
  - На каждом ребре появляется число (вес, 1-100)
  - Клик на число → инпут для редактирования веса
  - Enter применяет новый вес
  - Алгоритм Дейкстры пересчитывает кратчайший путь

Выбор пути:
  - Клик на узел 1 → SOURCE (зелёная рамка)
  - Клик на узел 2 → DESTINATION (синяя рамка)
  - Кратчайший путь подсвечивается #00e676
  - Остальные рёбра opacity 0.2
  - Над путём: "МАРШРУТ OSPF | ВЕС: 47"

Обрыв канала:
  - Long press на ребро 500мс → ребро обрывается
  - Становится красным пунктиром
  - Пакеты через него дропаются с анимацией ✕
  - OSPF пересчитывает путь в обход
  - Лог: "⚠ LINK DOWN: Router→Switch | OSPF RECONVERGE: 320ms"
  - Клик на оборванное ребро → восстанавливается

Индикатор:
  При активном OSPF вверху: "OSPF::ACTIVE | AREA 0 | МЕТРИКА: COST"
```

## 11. СЦЕНАРИИ (15 штук)

Кнопка [SCENARIOS] открывает меню выбора.
Каждый сценарий — интерактивная пошаговая история.

```
СЦ.01: "Первый запрос" — DNS резолюция
  Шаг 1: User набирает google.com
  Шаг 2: Запрос идёт к DNS-Stub
  Шаг 3: DNS-Stub → DNS-R → ROOT → TLD → AUTH
  Шаг 4: Ответ возвращается — IP адрес
  Шаг 5: Теперь TCP соединение к этому IP
  Обучение: как работает DNS

СЦ.02: "ТСПУ блокирует" — блокировка по IP
  User пытается зайти на blocked.com
  ТСПУ видит IP назначения → блокирует
  Объяснение: что именно ТСПУ читает в пакете
  Задание: почему DNS тоже не помогает

СЦ.03: "Поднять VPN" — обход блокировки
  Пошагово: выбор VPN сервера
  Показать инкапсуляцию: пакет в пакете
  ТСПУ видит только IP VPN сервера
  Пакет проходит

СЦ.04: "TCP рукопожатие" — SYN/SYN-ACK/ACK
  Анимация трёх пакетов туда-обратно
  Объяснение каждого шага
  Сравнение с UDP (нет рукопожатия)

СЦ.05: "OSPF находит путь" — алгоритм Дейкстры
  Интерактив: меняй веса рёбер
  Смотри как меняется маршрут
  Обрыв канала → автоперестройка

СЦ.06: "BGP hijack" — кража маршрута
  Сосед анонсирует твой префикс
  Трафик уходит не туда
  Задание: исправить через более специфичный анонс

СЦ.07: "DNS отравление" — подмена записей
  ТСПУ подменяет DNS ответ
  User получает неверный IP
  Задание: включить DoH (DNS over HTTPS)

СЦ.08: "Инкапсуляция" — матрёшка протоколов
  Пошагово показать L2→L3→L4→L7
  Что каждый узел видит и не видит
  Интерактив: нажать на слой чтобы развернуть

СЦ.09: "DDoS атака" — перегрузка канала
  Резкий рост трафика
  Канал краснеет (utilization 100%)
  Пакеты дропаются
  Задание: настроить rate limiting

СЦ.10: "CDN кэширование" — ускорение доступа
  Первый запрос: Cache MISS → идёт к origin серверу
  Второй запрос: Cache HIT → ответ за 5мс
  Сравнение задержки

СЦ.11: "TCP vs UDP" — выбор протокола
  Три потока: видео / файл / онлайн-игра
  Канал с 15% потерями
  Задание: выбрать правильный протокол
  Наблюдать что происходит при неправильном выборе

СЦ.12: "Обфускация трафика" — VLESS vs WireGuard
  ТСПУ распознаёт WireGuard по сигнатуре
  Блокирует
  Задание: переключиться на VLESS (маскируется под HTTPS)
  Пакеты проходят

СЦ.13: "DNSSEC" — подпись DNS записей
  Атака: DNS спуфинг
  Обычный DNS — уязвим
  DNSSEC — подпись проверяется → атака не проходит
  Интерактив: включить/выключить DNSSEC

СЦ.14: "BGP пиринг" — соединение AS
  Строишь BGP сессию с соседней AS
  Обмениваешься таблицами маршрутов
  Настраиваешь фильтры (что анонсировать, что принимать)

СЦ.15: "Полный маршрут" — от User до Server и обратно
  Полный цикл: DNS + TCP + HTTP + ответ
  Все узлы участвуют
  Инкапсуляция на каждом хопе
  Итог: сколько миллисекунд занял весь путь
```

### Формат сценария

```typescript

  id: number
  text: string                   // что объясняем
  highlightNodes: string[]       // какие узлы подсветить
  highlightEdges: string[]       // какие рёбра подсветить
  animatePacket?: {              // опциональная анимация пакета
    from: string
    to: string
    type: PacketType
  }
  choices?: Choice[]             // варианты выбора для игрока
  correctChoice?: number         // правильный вариант
  explanation: string            // объяснение после выбора
  waitForUser: boolean           // ждать нажатия "далее"
```

## 12. РЕЖИМ SANDBOX (Фаза 2)

Чистый холст для построения собственной сети.

### Панель инструментов (левая колонка)

```
Drag узла из панели на холст = создать узел
Клик + клик = создать ребро между узлами
Правый клик = удалить элемент
Двойной клик на ребро = редактировать параметры
```

### Три роли

```
[ЗА ПОЛЬЗОВАТЕЛЯ]
  Цель: достучаться до заблокированного сайта
  Инструменты: VPN, DoH, Shadowsocks
  Победа: пакет дошёл до WS3

[ЗА ТСПУ]
  Цель: заблокировать весь трафик к WS3
  Инструменты: IP-блокировка, SNI-фильтрация, DNS-подмена
  Победа: 0 пакетов к WS3 за 60 секунд

[ЗА VPN-ПРОВАЙДЕРА]
  Цель: помочь пользователям обойти ТСПУ
  Инструменты: VPN серверы, протоколы, чистые IP
  Победа: максимум туннелированных пакетов
  Экономика: биты с каждого туннелированного пакета
```

### Физика расстояний

```
Расстояние < 100px  → метка "LAN", задержка 0.1мс
Расстояние 100-400px → метка "MAN", задержка 5мс
Расстояние > 400px  → метка "WAN", задержка 40мс+
(только визуальное обучение, не влияет на геймплей)
```

## 13. ЭКОНОМИКА (Фаза 3)

Активируется после прохождения 5 сценариев.

```
БИТЫ (основная валюта):
  + за каждый доставленный пакет
  + за прохождение сценариев
  + пассивно от CDN кэш-хитов
  - на покупку узлов в Sandbox
  - на аренду чистых IP

ЧИСТЫЕ IP (редкая валюта):
  + за ивенты
  + покупка за биты (дорого)
  Нужны для: VPN серверов, обхода DPI

УРОВНИ AS:
  Личная AS → Региональный ISP → Федеральный → Магистральный → IXP
```

```
Режим веток:
  Янтарная рамка = экспериментальная ветка
  Зелёная рамка = main (стабильно)
  Красная рамка = конфликт при мерже

  > save "настроил OSPF"

  > merge → main
```
